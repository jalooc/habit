# Orbit + Ink — Design System

A small, opinionated design system for a **rotating-habits app**. The product surfaces *one habit at a time* from a *group* of related practices that cycle on a cadence (daily, weekdays, every-other-day, etc). Tick it off → the next one comes up tomorrow.

This system is **single-product, mobile-first** (Android Pixel-class, 412 × 892), calm and literary in voice, warm in tone. The defining visual is the **orbit ring** — a circle of dots where the up-next habit is always anchored at 12 o'clock and the ring rotates one notch when a habit is marked done.

## Product context

The original problem: managing several "rotational" practices in parallel — five different ab exercises, three languages, four people to call — is awkward on paper and on regular to-do or habit apps. Orbit makes the rotation itself the model: a **Group** holds an ordered queue of **Habits**, a cursor points at up-next, and any tick advances the cursor (or, if the user picks a different habit, moves that one to the back of the queue).

See `shared-data.js` for the data model and the four rotation helpers (`upNext`, `tickOff`, `skipTurn`, `tickOutOfOrder`). Port these verbatim — they encode the entire mechanic.

## Porting to React Native / Expo

**This system is authored for a native target.** The reference lab is web (HTML + inline JSX + CSS variables), but the canonical consumer is a **React Native / Expo** app: Expo SDK 56+, RN 0.85 New Architecture, **Unistyles 3** for styling, **React Navigation 7** (native-stack + formSheet), Legend State + MMKV. Read the rest of this README through that lens, and prefer `design-tokens.json` (named for RN/Unistyles conventions) over the CSS file when they disagree.

What that means in practice:

- **Styling is Unistyles**, not CSS. Define a theme and write `StyleSheet.create(theme => ({ … }))`. The two themes (Bone / Evening) map to Unistyles' adaptive light/dark; flips are live.
- **Token names match the theme**, not the old web vars. Use `background / surface / text / textSecondary / textTertiary / accent / accentDim / accentSubtle / border` (see Palette). The legacy web names (`bg / card / ink / …`) survive only in `colors_and_type.css` for the web lab; `design-tokens.json` carries the full old→new map under `legacyNames`.
- **No DOM, no hover, no `cursor`, no CSS.** Press feedback is `opacity: 0.6` via `Pressable`. There is no hover state to design.
- **Shadows are a single `boxShadow` string** per theme (RN 0.81+ / Unistyles accept the shorthand), not layered CSS box-shadows conceptually — see Elevation.
- **Fonts are discrete embedded TTFs**, not Google `@import` and not variable fonts — see Type.
- **Icons are Lucide** (`lucide-react-native`), not Unicode glyphs — see Iconography.
- **The orbit ring is Views + Reanimated**, not SVG — see Animation and `components.md`.
- **Navigation/sheets** follow the project's `CLAUDE.md`. One hard native rule: **never stack sheets** — expand in place or flip a single sheet between modes (reader → editor in the *same* sheet).
- **Interaction is slide/swipe to act, tap to open** — see Interaction model below. (This replaces the old "tap any row to tick".)

## Content fundamentals

**Voice.** Warm, literary, slightly slow. Sentences are short and observational. The app talks like a thoughtful friend, not a coach. Never cheerleader-y; never gamified. No streaks-as-screaming; no "Crush it today!"; no emoji except the rare ✓ inside a coral dot.

**Casing.** Sentence case for UI strings; uppercase + letter-spacing for kicker labels ("UP NEXT", "THE QUEUE", "NOW"). Group names are user-written and rendered in italic Fraunces.

**Pronoun.** Indirect — "the ring waits", "the next ping comes when it's time". The user is not addressed as "you" except where it's earned ("Your due rotations are all squared away.").

**Time language.** Avoid "day" as a measurement of cadence (a rotation can ping in minutes). Use "a moment", "right now", "when it's time". The cadence labels themselves *do* use day language because they refer to the user's chosen schedule ("Once a day", "Weekdays", "Every 2 days") — that's their meaning.

**Examples.**

| Bad | Good |
|---|---|
| "Crush your routine today!" | "Today · 4 rotations" |
| "Streak: 11 days 🔥" | "Streak · 11 days" |
| "You missed yesterday" | (silent — habit carries over) |
| "Skip" | "Skip this turn" / "The habit cycles to the back of the ring, untouched." |
| "No rotations due today" | "A quiet moment — nothing in rotation." |
| "Daily reminder" | "Reminder when due" |
| "Quiet hours" | "Active hours" |

A "Literary" and a "Functional" tone dictionary live in `tokens.jsx` (the Tweaks panel can toggle between them). The Literary set is the default; Functional exists for accessibility users / shorter copy and is *not* a separate brand voice — it's the same calm tone, just tighter.

See `microcopy.md` for the full dictionary.

## Visual foundations

### Palette

**One chromatic accent.** Coral (#c96442) on light, sand-coral (#e3a37a) on dark. Used for:
- the active dot on the orbit ring
- "UP NEXT" / "NOW" kickers
- the coral chip ("5 habits in rotation", "Active hours · 8:00 – 22:00")
- the prominent **Rotation paused** banner
- the wordmark "◯ ORBIT"

The primary CTA is **always ink-on-bone** ("MARK DONE" pill). Coral is never used for a primary action — it stays close to the rotation itself.

Two themes, no more:
- **Light · Bone** — `#f5f1ec` bg, `#ffffff` cards, `#1a1814` ink. The warm default.
- **Dark · Evening** — `#1c1d20` bg, `#252629` cards, `#ece8e2` ink. Sand-coral accent.

Earlier exploration included Moss and Slate variants. Those are gone — Bone and Evening are the design system.

**Token names (native).** The theme keys are `background`, `surface`, `text`, `textSecondary`, `textTertiary`, `accent`, `accentDim`, `accentSubtle`, `border`. These are the names to use in a Unistyles theme. The web lab still uses the legacy CSS vars (`--bg`, `--card`, `--ink`, `--ink-soft`, `--ink-faint`, `--accent-soft`, `--rule`); `colors_and_type.css` now also exposes the canonical names as aliases, and `design-tokens.json` carries the full map under `legacyNames`.

### Type

**Fraunces + Inter.** Two families.

| Role (token) | Family (PostScript name) | Why |
|---|---|---|
| Display, group names, body (`title` / `heading` / `body`) | Fraunces-Regular (+ Fraunces-Italic for group names) | Warm; reads "literary". |
| Kickers, buttons, meta, labels (`label` / `button` / `caption`) | Inter-Regular / Inter-Medium | Sharp; reads "system"; lets Fraunces carry the personality. |

Avoid Helvetica, Arial, Roboto, system-ui-default for body copy.

**Native font rules (this is reality, not aspiration):**

- **Embed discrete TTFs** via `expo-font` / `useFonts`. **No Google Fonts `@import`** (that's web-only) and **no variable-font axes** — the `opsz` axis can't be relied on across platforms, so each type role uses a **fixed size** (see `design-tokens.json` → `type`). The web lab keeps the variable import for fidelity; native does not.
- **`fontFamily` must equal the PostScript name** of the embedded file — not `"Fraunces"` plus a `fontWeight`. The four faces we ship are exactly: `Fraunces-Regular`, `Fraunces-Italic`, `Inter-Regular`, `Inter-Medium`.
- **Only ship the weights you reference.** We bundle Fraunces 400 + italic and Inter 400/500. **Inter 600/SemiBold is dropped** — any unbundled weight renders as synthetic **faux-bold on Android**. If you genuinely need it, bundle `Inter-SemiBold` explicitly.
- **Caveat — italic times.** Italic Fraunces at 13–14px silently drops trailing colon-digit runs (e.g. "9:09" loses the ":09") on Android — a shaping bug. **Never set clock times in italic;** use `caption` (Inter-Regular) or upright Fraunces for times.

See `colors_and_type.css` for the web semantic styles (`.oi-h1`, `.oi-kicker`, etc.) and `design-tokens.json` for the canonical type token map (role names: `title`, `heading`, `body`, `caption`, `label`, `button`; `ringCenter` and `buttonSmall` are marked optional — no current app equivalent).

### Spacing & shape

Six radii, used systematically. The token names are a flat T-shirt scale (the old semantic names `hero/card/inner/subtle/input` collided with component names — the intent is kept as a note):

| Radius (token) | Value | Use |
|---|---|---|
| `xl` | 28 | Hero cards (the rotation ring lives in one of these) |
| `lg` | 22 | List cards (Rotations index) |
| `md` | 18 | Secondary cards, inputs blocks, queue list |
| `sm` | 16 | Inner clusters, settings rows |
| `xs` | 14 | Single inputs (name field) |
| `pill` | 100 | All buttons, chips, badges, slide-to-confirm track |

**Spacing is a single flat scale** (native apps don't carry a density mode — the old compact/cozy/roomy model is gone):

`xs` 4 · `sm` 8 · `md` 12 · `lg` 16 · `xl` 20 · `2xl` 24 · `3xl` 32 · `4xl` 48

The old "cozy" defaults map onto it: screen padding → `xl` (20, rounded from 22) · card padding → `lg` (16) · stack gap → `md` (12) · hero ring box → top `2xl` (24) / sides `lg` (16) / bottom `xl` (20) · bottom safe area → `3xl` (32) below the last block (plus the device safe-area inset).

### Backgrounds, gradients, imagery

**No gradients in the app body.** A single exception: the **lock-screen notification preview** uses a coral→ink linear gradient to evoke a phone wallpaper. Everywhere else: flat warm bone, flat dark slate.

**No imagery.** No photos, no hand-drawn illustrations, no full-bleed graphics. The orbit ring (SVG, generated) and a few Unicode marks are the only "imagery". If a screen feels empty, that's intentional — the literary voice fills the space, not graphics.

### Elevation

**One shadow.** `--shadow` (token `shadow.card`) — used on all cards. Express it as a **single `boxShadow` string per theme** (RN 0.81+ / Unistyles accept the CSS-style shorthand); don't reach for `elevation` or `shadow*` prop soup. Paused list cards drop the shadow entirely and lower opacity to 0.7. Hero on paused detail screens stays elevated but desaturated.

```
0 1px 2px rgba(26,24,20,0.04), 0 8px 32px rgba(26,24,20,0.08)   /* Bone  */
0 1px 2px rgba(0,0,0,0.25),    0 12px 36px rgba(0,0,0,0.4)      /* Evening */
```

**Android caveat.** Keep the ambient layer's reach **tight** relative to the inter-card gap. Android renders `boxShadow` as a flat plateau, not a true gaussian falloff — if the blur radius is large compared to the gap between cards, adjacent shadows merge and flood the gap into a solid gray band. The 32px ambient blur above is tuned against the cozy card gap (12–16px); widen one and you must widen the other.

The paused **banner** uses its own coral-tinted glow (`shadow.paused`).

### Animation

**One headline motion: the ring rotates one notch when a habit is marked done.**

```
duration: 550ms
easing:   cubic-bezier(.55, .06, .3, 1)
follow-up: cursor advances at 600ms; tab pager nudges to next due group ~250ms after that.
```

The active dot has a slow halo pulse (`r: 14 → 20 → 14`, opacity `.16 → .06 → .16`, 2.4s, infinite).

All other UI motion is short ease (200ms): button opacity, tab swap, chip width changes (the pager dot elongates to 18×6), toggles.

**Native:** drive the ring settle and halo with **Reanimated** — `withTiming(target, { duration: 550, easing: Easing.bezier(.55,.06,.3,1) })` for the one-notch settle, `withRepeat(withTiming(…), -1, true)` for the halo pulse. The slide-to-confirm knob uses a gesture-driven shared value that springs back on incomplete release.

**No bouncy springs. No micro-celebrations.** A tick lands quietly.

### Press, focus (native) · hover (web only)

- **Press** (the native default): opacity 0.6, no scale. Use `Pressable`.
- **Hover / `cursor`**: **web only.** Native has no hover state and no cursor — don't design one. (The web lab uses opacity 0.85 on hover.)
- **Focus-visible**: web only — 1.5px coral outline + 2px offset. Native relies on platform focus.
- **Disabled**: opacity 0.5 (disabled `Pressable`).

### Layout rules

- Mobile-first, 412 px design width. Everything scales for larger viewports by sitting in the center (the lab clamps the prototype to 412 × 892 with a device frame).
- Tab bar at the bottom (4 tabs: Today / Rotations / Insights / Settings).
- Status strip at the top inside the device chrome: `◯ ORBIT` left, weekday + date right.
- `xl` (20) horizontal padding (the old cozy 22, rounded to the scale).
- Bottom safe area: `3xl` (32) below the last content block, plus the device safe-area inset.

### Interaction model — slide/swipe = act, tap = open

Ticking is destructive and hard to undo, so it is never a bare tap. (This **replaces the old "tap any row to tick"** rule.)

- **Hero · up-next habit:** ticked with a **slide-to-confirm** control, with a ghost **"Skip this turn"** beside it. The slider is the hero CTA — it replaces the old "MARK DONE" pill. (Ink-on-bone pills remain the pattern for *real* buttons: Continue, Create, sheet Save.)
- **Queue rows:** tick **out of order via swipe-right** over a coral pane. Rows have **no skip** — skip only means something for the current turn.
- **Tap opens:** tapping a habit (row or list card) opens its read/edit sheet. Image thumbs tap-open a viewer. Tapping never ticks.
- **Long-press is reserved** (group title → edit) and fires on **hold-elapsed**, not on release.

See `components.md` (Hero card, Queue list item) for the full spec.

## Iconography

**Lucide is the official icon set** (`lucide-react-native` on native, `lucide` / inline SVG on web), drawn at **`stroke-width: 1.5`** to match the system's thin, calm line. Earlier the system said "resist Lucide" — that isn't viable for a real app, so Lucide is now canonical. Keep **Unicode only for true typographic marks** inside text (`· ¶ — ↗`).

**Glyph → Lucide mapping:**

| Old glyph | Use | Lucide |
|---|---|---|
| `◯` U+25EF | Wordmark mark (“◯ ORBIT”), inactive-rotation tab | `circle` (the wordmark ring may stay a drawn `View` circle for crispness) |
| `◐` U+25D0 | "Today" tab | `disc` |
| `◔` U+25D4 | "Insights" tab | `bar-chart-2` |
| `◌` U+25CC | "Settings" tab | `settings` |
| `✓` U+2713 | "done" mark in the active dot / swipe pane | `check` (inside a coral circular `View`) |
| ring/queue separators | between dots / segments | drawn `View`s, not a glyph |

**Kept as Unicode (typographic marks in copy):**

| Glyph | Use |
|---|---|
| `·` U+00B7 | Separator in kickers + meta ("Once a day · 5 habits"). |
| `—` U+2014 | Em-dash in copy ("A quiet moment — nothing in rotation."). |
| `↗` U+2197 | Inline "opens" / external affordance in running text. |

**No emoji**, except the coral-circle `✓` (now a Lucide `check` in a coral dot).

## Working with this system in Claude Code

The fastest path:

1. Read this README and `microcopy.md`.
2. **Build a Unistyles theme** from `design-tokens.json` (two themes: Bone / Evening, adaptive). Use the canonical token names (`background`, `surface`, `text`, …). For a *web* mock instead, import `colors_and_type.css` once at the root and flip with `data-theme="dark"`.
3. Port `shared-data.js` verbatim — the four rotation helpers are the entire mechanic.
4. For visual recreation, use the screen files as reference. They're inline-JSX (no build step) and self-contained; pull patterns into React Native (Unistyles `StyleSheet.create(theme => …)`), Compose, or plain React.
5. For the orbit ring: position dots at `(i / n) * 360 - 90` degrees, rotate the whole group by `-cursor * (360/n)`, counter-rotate each dot's content so it stays upright. On tick, animate `-step` over 550 ms with `cubic-bezier(.55,.06,.3,1)`. **Native:** build it from `View`s + transforms with Reanimated (see `components.md` → `OrbitRing` for the recipe); SVG is optional. Math source is `rings.jsx`'s `OrbitRing`.
6. **Sheets never stack** (iOS formSheet limitation) — expand in place or flip the same sheet between modes. Pick the right React Navigation presentation per screen (project `CLAUDE.md`).

## Caveats

- **Fonts are discrete embedded TTFs on native** (`expo-font`), with `fontFamily` = PostScript name (`Fraunces-Regular`, `Fraunces-Italic`, `Inter-Regular`, `Inter-Medium`). No variable axes; **Inter 600 is not shipped** (faux-bold on Android if referenced); **don't set times in italic** (Android drops colon-digit runs at 13–14px). The web lab still loads Fraunces + Inter from Google via `@import` in `colors_and_type.css` — that's web-only.
- **Icons are Lucide** (`stroke-width: 1.5`). Unicode is reserved for typographic marks in copy (`· — ↗`). See Iconography.
- **No imagery assets in the core system.** The orbit ring + Lucide marks are the only graphics; the per-habit *images* feature (`images.jsx`) is user-supplied content, not brand imagery. If your product needs photography or illustration, that's a separate decision — flag it.
- **Mobile-only.** No tablet or desktop layouts here. The lab is 412 × 892; real builds use device safe-area insets.
- **Sheets never stack.** Don't present a sheet from a sheet (broken on iOS formSheet) — expand in place or flip one sheet between modes.
